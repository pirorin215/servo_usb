# Servo USB Controller

Pro MicroとSG90サーボモーターを使用したUSB制御スイッチシステム。赤外線学習機能により、スマートリモコンとの連携が可能です。

## ⚠️ 重要な制限事項

**このデバイスはNECプロトコル専用に設計されています。**

- ✅ 対応プロトコル: **NEC**（日立、東芝、Canon、他多数の日系メーカー）
- ❌ 非対応プロトコル: パナソニック、ソニー（SIRC）、その他独自プロトコル

### スマートリモコンの設定について

学習させる信号は、スマートリモコンで**必ずNECプロトコルのメーカー**を選択してください：

- 推奨メーカー: **Hitachi（日立）**, **Canon**, **Toshiba（東芝）**
- その他NEC準拠のメーカー

スマートリモコンのメーカー設定で、パナソニックなどを選ぶと、信号を受信できても警告が出て学習できません。

## 機能

- **サーボモーター制御**: SG90サーボモーターで物理的なスイッチ操作を自動化
- **オルタネートスイッチ対応**: ON/OFFの2状態をスムーズに切り替え
- **赤外線学習機能**: スマートリモコンの信号を学習して保存（最大3種類：ON/OFF/PLAYPAUSE）
- **プロトコル照合とRAW波形照合のハイブリッド**: 高精度な信号認識
- **EEPROM保存**: 学習した信号を不揮発メモリに保存（電源切断後も保持）
- **赤外線送信**: 学習した信号を送信してスマートリモコン操作
- **HIDキーボード/Consumer機能**: ディスプレイウェイクアップや再生/一時停止など
- **ブザー音フィードバック**: 操作状態を音で確認
- **低電力モード**: サーボをデタッチして消費電力を削減

## 必要なハードウェア

| コンポーネント |仕様|
|---|---|
| マイコン | Arduino Pro Micro (ATmega32U4) |
| サーボモーター | SG90 (または互換品) |
| スイッチ | オルタネートスイッチ |
| 赤外線LED | 赤外線送信用LED |
| 受信モジュール | VS1838Bまたは同等の赤外線受信モジュール |
| ブザー | 圧電ブザー |

## ピン接続

| Pro Microピン | 接続先 |
|---|---|
| D5 | サーボモーター信号線(オレンジ/黄) |
| D4 | オルタネートスイッチ |
| D3 | 赤外線LED (+抵抗直列) |
| D2 | 赤外線受信モジュール信号 |
| D6 | ブザー |
| VCC | サーボ赤線、スイッチ側 |
| GND | サーボ茶線、ブザー、受信モジュールGND |

### 回路図の注意点

- 赤外線LEDには必ず適切な抵抗（約100〜330Ω）を直列接続してください
- サーボモーターの電源はPro MicroのVCCから供給可能ですが、複数接続する場合は外部電源を検討してください

## 必要なライブラリ

Arduino IDEのライブラリマネージャから以下をインストールしてください：

- **Servo** (Arduino標準ライブラリ)
- **IRremote** by Armin Joachimsmeyer (v3.9.0以上)
- **HID-Project** by NicoHood (v2.8.0以上) - Keyboard & Consumer制御用
- **EEPROM** (Arduino標準ライブラリ)

## インストール手順

1. Arduino IDEを開く
2. `ツール` > `ボード` > `Arduino AVR Boards` > `Arduino Pro Micro` を選択
3. `ツール` > `シリアルポート` でPro Microのポートを選択
4. `servo_usb.ino` を開く
5. 必要なライブラリをインストール
6. 検証ボタン(✓)をクリック
7. アップロードボタン(→)をクリック

## 使い方

### 初期設定（必須）

このシステムを使用する前に、スマートリモコンの信号を学習させる必要があります。

1. Arduinoにスケッチをアップロード
2. 以下のコマンドで信号を学習:

```bash
./ir_learn.sh on   # スマートリモコンのONボタンを学習
./ir_learn.sh off  # スマートリモコンのOFFボタンを学習
```

学習が完了すると、EEPROMに信号が保存されます。

### 基本操作

1. **USB接続時**: ピッという短いブザー音が鳴り、起動を知らせます
2. **スイッチ操作**: 物理スイッチを操作すると、サーボが動作してスイッチを切り替えます
3. **赤外線受信時**:
   - ON信号受信: ピーー（長い音）→ サーボが動作
   - OFF信号受信: ピッピッ（2回の短い音）→ サーボが動作
   - PLAYPAUSE信号受信: HDMI-CECまたはConsumerキーで再生/一時停止を送信

### 赤外線信号学習と送信

このシステムはスマートリモコンの赤外線信号を学習して使用します。

#### 学習方法

**シェルスクリプトを使用する場合（推奨）:**

```bash
# ON信号を学習
./ir_learn.sh on

# OFF信号を学習
./ir_learn.sh off

# PLAYPAUSE信号を学習（再生/一時停止など）
./ir_learn.sh playpause

# 学習したパターンを確認
./ir_learn.sh dump

# 学習データをリセット
./ir_learn.sh reset
```

**シリアルモニタを使用する場合:**

1. シリアルモニタを9600 baudで開く
2. 以下のコマンドを入力:
   - `LEARN_ON` - ON信号の学習を開始
   - `LEARN_OFF` - OFF信号の学習を開始
   - `LEARN_PLAYPAUSE` - PLAYPAUSE信号の学習を開始
   - `RESET_PATTERNS` - 学習データを全てリセット
   - `DUMP_PATTERNS` - 学習したパターンを表示

3. 学習モード中（ピッ鳴った後）にスマートリモコンのボタンを押す
4. 学習成功するとピッピッ鳴る

#### 対応しているスマートリモコン

- プロトコル: NEC形式
- 周波数: 38kHz
- アドレス: 0x82
- ONコマンド: 0x26
- OFFコマンド: 0x3E

※ 異なるアドレス/コマンドのスマートリモコンも学習可能です

## 技術的な特徴

### 赤外線認識方式

このシステムは2段階の認識方式を採用しています：

1. **プロトコル照合（高速）**: NECプロトコルの場合、アドレスとコマンドで即座に識別
2. **RAW波形照合（高精度）**: プロトコルが不明または照合失敗時、波形パターンで識別

これにより、既知のスマートリモコンは瞬時に反応し、未知のリモコンも高精度で認識します。

### 効率化の取り組み

- **遅延ロード**: プロトコル照合で必要な情報のみをEEPROMから読み取り（最大66%削減）
- **イベントドリブン**: dump/resetコマンドで完了待機を動的に判断（平均待機時間80-90%短縮）
- **コード重複排除**: ヘルパー関数で127行の重複コードを削減

### デバッグモード

スケッチ内の `DEBUG_MODE` 定数を `1` に設定すると、詳細なデバッグ出力が有効になります：

```cpp
#define DEBUG_MODE 1  // 0:無効, 1:有効
```

デバッグ時はIR受信の詳細情報（プロトコル、アドレス、コマンド、RAW波形など）がシリアルモニタに表示されます。

## トラブルシューティング

### 学習した信号で動作しない
- `./ir_learn.sh dump` で学習データが正しく保存されているか確認してください
- スマートリモコンをArduinoの受信モジュールに向けて（5〜10cm程度）再度学習してください
- EEPROMが破損している可能性があります。`RESET_PATTERNS` コマンドでリセットしてください

### 赤外線学習ができない
- Arduinoとスマートリモコンの距離を5〜10cmに近づけてください
- 受信モジュールの配線を確認してください
- 他のリモコン（テレビなど）で受信できるかテストしてください

### サーボが動かない
- サーボの信号線が正しく接続されているか確認してください
- 電源が不足している場合、外部電源の使用を検討してください
- 学習が完了しているか確認してください（EEPROMが空の場合は送信しません）

### スイッチ操作で反応しない
- スイッチの配線を確認してください
- シリアルモニタでスイッチ状態が正しく読めているか確認してください

### HIDキーボード/Consumerが動作しない
- Pro Microを一度再接続してください
- シリアルモニタで「HID Keyboard sent」または「HID Consumer sent」と表示されているか確認してください
- HID-Projectライブラリが正しくインストールされているか確認してください
